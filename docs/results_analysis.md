# Анализ результатов

## Основные результаты

| Метод | Симуляция (%) | Реальность (%) | Потеря (п.п.) |
|:---|:---:|:---:|:---:|
| ПИД-регулятор | 72 | 70 | 2 |
| DRL (PPO) без адаптации | 94 | 78 | 16 |
| DRL (PPO) + Domain Randomization | 91 | 89 | 2 |
| DRL (SAC) + Domain Adaptation | 93 | 91 | 2 |

## Интерпретация

### Разрыв Sim-to-Real
DRL-алгоритм PPO демонстрирует отличную точность в симуляции (94%), однако при переносе на реального робота показывает значительную потерю (16 п.п.). Это объясняется:
1. **Неточностями физической модели** - трение, инерция, упругость в реальном роботе отличаются от симуляции
2. **Сенсорным шумом** - реальные датчики имеют шумы, отсутствующие в симуляции
3. **Задержками** - реальные системы управления имеют латентность

### Эффективность Domain Randomization
Вариация параметров в процессе обучения (трение, массы, шум) позволяет агенту стать робастным к различиям между симуляцией и реальностью. Потеря точности сокращается с 16% до 2%.

### Комбинация SAC + Domain Adaptation
Наилучший результат достигнут при использовании алгоритма SAC (более стабилен для непрерывного управления) и дообучения на 50 реальных траекториях.

## Статистическая значимость
Все измерения проведены при 100 независимых испытаниях. Доверительный интервал 95% не превышает ±3% для всех измерений.
